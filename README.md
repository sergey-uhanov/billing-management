# Middle Node.js Assignment

## Опис проєкту

Цей проєкт — серверний застосунок, побудований на **NestJS**, який реалізує систему автентифікації користувачів, керування акаунтами та фінансові операції між користувачами з підтримкою ролей та webhook-інтеграції.

Система забезпечує:
- реєстрацію та логін користувачів
- JWT-автентифікацію (access + refresh tokens)
- рольову модель доступу (Admin, Client)
- керування акаунтами та балансами
- перекази між користувачами
- обробку транзакцій
- відправку подій у webhook
- Docker-орієнтоване розгортання

---

## Технології

- **Backend**: NestJS
- **Database**: PostgreSQL
- **ORM**: TypeORM (з підтримкою міграцій)
- **Authentication**: JWT (access/refresh tokens)
- **API Documentation**: Swagger (OpenAPI)
- **Containerization**: Docker, Docker Compose

---

## Основний функціонал

### Аутентифікація
- Реєстрація користувача з email та password
- Авторизація з видачею JWT токенів
- Зберігання refresh token у cookies
- Автоматичне блокування доступу для деактивованих акаунтів

### Рольова модель
- **Admin**
    - перегляд користувачів
    - блокування акаунтів
    - перегляд та скасування транзакцій

- **Client**
    - створення акаунтів
    - перегляд балансу
    - створення транзакцій
    - поповнення балансу
    - перегляд власних транзакцій

---

## Фінансові операції
- переказ коштів між користувачами
- контроль балансу
- атомарні транзакції (через database transactions)
- можливість скасування транзакцій
- синхронізація стану балансу

---

## Webhook інтеграція
Після створення або зміни транзакції система відправляє дані у зовнішній webhook endpoint (наприклад, webhook.site).

---


## Сидінг бази даних
При старті застосунку:
- створюються ролі (Admin, Client)
- створюється дефолтний admin користувач

---

## Swagger документація
API повністю задокументовано через Swagger:
- опис всіх endpoint'ів
- схеми запитів і відповідей
- приклади помилок
- типи DTO

Після запуску доступно:
```http://localhost:4000/api/docs```

## Запуск проєкту

### 1. Клонування репозиторію
```bash
git clone https://github.com/sergey-uhanov/billing-management
cd billing-management
```

### 2. Налаштування змінних середовища
Створіть файл .env у корені проєкту та заповніть його .env.example


### 3. Запуск через Docker Compose
```bash
docker-compose up
```
### 4. Перевірка роботи сервісу
Після запуску застосунок буде доступний:

API: http://localhost:4000
Swagger документація: http://localhost:3000/docs

## Архітектура проєкту

Проєкт побудований на модульній архітектурі NestJS:

- Auth Module — автентифікація та JWT
- User Module — користувачі та ролі
- Account Module — акаунти та баланси
- Transaction Module — фінансові операції


## Примітки
- Усі критичні операції (перекази, зміна балансу) виконуються в транзакціях БД
- Деактивовані користувачі не можуть отримувати доступ до системи
- Всі фінансові дії логуються через transaction entity
- Webhook викликається асинхронно після завершення операції
